Zero to Hero

Entering into cybersecurity is challenging and can be a pain in the ass !!

To start with, here are my 2 cents which can help you to start and get the basic idea about how to, where to and what to start in this career !!

Phase 1

  1. Networking
    1. Networking is very important yet a little bit tricky but once you master it, it will be a game changer for you.
    1. All you need to know about IP address, subnetting, ports, well known ports, services, firewall.
    1. Till date, I am using these two channels to cover up basics
      1. Neso academy (bit older but useful)
        1. Neso Academy
      1. OSI model
        1. Youtube video
  1. Start with Linux, DO NOT go with any fancy cyber security OS. Start installing the tools by your own. Which tools? Those which are being used in penetration testing of web, mobile, network and API.
  1. Build some bash scripting, take help from internet. How to make one and how to run the same.
    1. Build a habit of commit all your content to the github Repo to maintain your work and be always ready to show it to world.
  1. Once you done with the above, start developing a CURD operation website.

Phase 2

You are all set to start for entering to the security world.

Will start will the web penetration testing. Since, website have to sides

  • Client side
  • Server side

If you think as a developer mindset, you are developing a feature that have both client and sever ans performing a CURD operation.

But

If you are thinking as a security engineer, you are understanding the workflow, how operation are being perform and what can be the possible vulnerability can be there.

To understand the different type of vulnerabilities and their impact. You can always follow the lab called Portswigger Academy

Link - https://portswigger.net/web-security

Phase 3

Since, the whole world is shifting to the cloud the risk of the cloud is also increased. Well in terms of securing the cloud infrastructure, you need to be excel in the same.

There are few very well known cloud service providers which are named as

  • AWS ( Amazon web service)
  • Azure
  • GCP ( google cloud provider)
  • Linod

etc

So there is a website which is more of an Encyclopaedia, that helps you to gain knowledge of security aspects of the cloud infrastructure.

You can find the link here - https://hackingthe.cloud/
