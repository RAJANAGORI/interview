# Building an AppSec Program - Interview Questions & Answers

## 60-second answer

**Q: You're the first AppSec hire—first 90 days?**

**A:** **Days 1–30:** asset/data inventory, incident history, secret scan + MFA quick wins, eng relationships. **Days 31–60:** risk tiers, CI SCA/secrets, threat modeling for Tier 1, security champions. **Days 61–90:** pilot release gate on one product, pen test/bounty pilot, metrics dashboard and roadmap. Partner with engineering—reduce risk without becoming a bottleneck.

---

### Q1: How do you measure program success?
**A:** **MTTR** for High+, **Critical in prod** trend, **% Tier-1** with threat models, **scanner signal ratio**, **repeat finding rate**, developer satisfaction with consult SLAs.

### Q2: Security champions—worth it?
**A:** Yes—**1 per squad** as force multiplier for triage, standards feedback, and local coaching; AppSec retains policy and deep reviews.

### Q3: Build vs buy for tooling?
**A:** Start with **platform defaults** (GitHub Advanced Security, cloud CSPM) plus **one** best-in-class per layer; expand when **signal** justifies cost.

---

## STAR prompt

*"Tell me about standing up AppSec at a fast-growing startup."* — Use 30/60/90 structure, quick wins, pilot gate, metrics.
