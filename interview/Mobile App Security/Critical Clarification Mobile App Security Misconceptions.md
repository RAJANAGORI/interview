# Critical Clarification — Mobile App Security Misconceptions

## 1. "App store review guarantees security."
**Reality:** Store review is not a full security assessment.

## 2. "Obfuscation stops attackers."
**Reality:** It slows analysis but does not prevent determined reverse engineering.

## 3. "Root/jailbreak checks solve tampering."
**Reality:** These checks are bypassable; use layered controls.

## 4. "Pinning alone prevents MITM risk."
**Reality:** Operational pin failures and backend weaknesses still matter.

## 5. "Client-side validation is enough."
**Reality:** Authorization and critical validation must be server-side.

## 6. "Encrypted local DB means no data risk."
**Reality:** Key handling and runtime compromise can still expose data.

## 7. "Mobile apps do not need threat modeling."
**Reality:** Mobile adds unique surfaces that require explicit modeling.

## 8. "One pentest per year is enough."
**Reality:** Release cadence requires continuous validation.
