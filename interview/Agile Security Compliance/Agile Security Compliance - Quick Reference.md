# Agile Security Compliance - Quick Reference

## **Key Strategies**

| Strategy | Description |
| --- | --- |
| **Automation** | Automated security and compliance checks |
| **Shift-Left** | Security early in SDLC |
| **Security as Code** | Policy and configuration as code |
| **Continuous Compliance** | Ongoing monitoring vs. point-in-time |

## **DevSecOps Integration**

| SDLC Phase | Security Activities |
| --- | --- |
| **Design** | Threat modeling, security architecture |
| **Development** | Secure coding, security libraries |
| **CI/CD** | Automated scanning, security gates |
| **Deployment** | Security checks, compliance verification |
| **Operations** | Continuous monitoring |

## **Security Automation Checklist**

- ✅ SAST in CI/CD
- ✅ DAST scanning
- ✅ Dependency scanning
- ✅ Container scanning
- ✅ Infrastructure scanning
- ✅ Compliance checks
- ✅ Policy enforcement

## **Compliance Frameworks**

| Framework | Description | Applicability |
| --- | --- | --- |
| **SOC 2** | Security, availability, confidentiality | SaaS companies |
| **ISO 27001** | Information security management | International |
| **PCI-DSS** | Payment card security | Payment processors |
| **HIPAA** | Healthcare data security | Healthcare |
| **GDPR** | Data protection | EU data processing |

## **Security as Code**

| Component | Implementation |
| --- | --- |
| **Policy as Code** | Security policies defined as code |
| **Infrastructure as Code** | Secure infrastructure templates |
| **Configuration as Code** | Security configurations versioned |
| **Compliance as Code** | Compliance checks automated |

## **Metrics**

| Metric Type | Examples |
| --- | --- |
| **Security** | Vulnerabilities found/fixed, remediation time |
| **Compliance** | Check pass rate, policy violations |
| **Process** | Security review participation, test coverage |

## **Key Principles**

- Automate security and compliance
- Integrate into CI/CD
- Continuous compliance monitoring
- Risk-based approach
- Security culture building
