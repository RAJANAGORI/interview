# Security vs Usability Balance - Quick Reference

## **Key Principles**

| Principle | Description |
| --- | --- |
| **Risk-Based** | Match security to risk level |
| **Contextual** | Adapt security to context |
| **Progressive** | Increase security as risk increases |
| **User-Centric** | Design security with users in mind |

## **Adaptive Authentication**

| Risk Level | Security Control |
| --- | --- |
| **Low** | Minimal friction (remember device, trusted location) |
| **Medium** | Moderate friction (MFA for new device) |
| **High** | Higher friction (re-authentication, MFA required) |

## **Security-Usability Trade-offs**

| Action | Risk Level | Security Control |
| --- | --- | --- |
| View profile | Low | None additional |
| Change settings | Medium | Re-authentication |
| Transfer funds | High | MFA required |
| Delete account | High | Multiple confirmations + MFA |

## **Friction Reduction Techniques**

- ✅ Single Sign-On (SSO)
- ✅ Password managers
- ✅ Biometric authentication
- ✅ "Remember this device"
- ✅ Progressive disclosure
- ✅ Security education

## **Best Practices**

| Practice | Description |
| --- | --- |
| **Make Security Invisible** | Transparent when possible |
| **Provide Options** | Multiple MFA methods |
| **Educate Users** | Explain security benefits |
| **Measure UX** | Security UX metrics |
| **Iterate** | Continuous improvement |

## **Common Solutions**

| Solution | Security | Usability |
| --- | --- | --- |
| **Biometrics** | High | High |
| **SSO** | Medium-High | High |
| **Password Managers** | High | High |
| **Hardware Keys** | Very High | Medium |

## **Key Takeaways**

- Security and usability can be complementary
- Risk-based approach reduces friction
- Technology can make security easier
- User education improves adoption
- Measure and iterate on security UX
